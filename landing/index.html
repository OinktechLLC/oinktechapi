Подключи за 5 минут

Никаких ключей, регистраций и лимитов. Просто форкни репо и используй данные.

01

Форкни репозиторий

Клонируй наш GitHub-репозиторий к себе. Это займёт 30 секунд.

github.com/YOUR_ORG/oinktech-api
02

Включи GitHub Actions

Actions уже настроены. Просто разреши их запуск в настройках форка — парсеры запустятся автоматически по расписанию.

03

Читай JSON напрямую

Данные коммитятся в репо как статичные файлы. Читай через raw.githubusercontent.com или деплой на GitHub Pages.

raw.githubusercontent.com/YOUR/oinktech-api/main/data/tv/schedule_latest.json
04

Встрой в своё приложение

Используй любой HTTP-клиент: fetch, axios, requests — всё что угодно. Смотри пример кода →

JavaScript
// Получаем расписание ТВ
const BASE = "https://raw.githubusercontent.com/YOUR/oinktech-api/main/data";

// Все каналы сегодня
const tv = await fetch(`${BASE}/tv/schedule_latest.json`)
  .then(r => r.json());

// Конкретный канал
const ntv = await fetch(`${BASE}/tv/channels/ntv.json`)
  .then(r => r.json());

// Погода в Москве
const weather = await fetch(
  `${BASE}/weather/cities/moscow.json`
).then(r => r.json());

// USD/RUB
const rates = await fetch(`${BASE}/rates/rates_latest.json`)
  .then(r => r.json());
const usd = rates.highlight.USD;

Часто задаваемые вопросы

Нет. OinkTech API работает полностью без ключей и регистрации. Данные хранятся как статичные JSON-файлы в открытом GitHub-репозитории и доступны через raw.githubusercontent.com или GitHub Pages без каких-либо ограничений.
Расписание ТВ — каждые 3 часа. Новости — каждые 6 часов. Погода и курсы валют — каждый час. Все обновления происходят через GitHub Actions по расписанию (cron) и коммитятся прямо в репозиторий.
Да, именно для этого сделан открытый реестр каналов в tv_parser.py и расширяемый список RSS-источников в news_parser.py. Форкни репо, добавь свои каналы или источники в соответствующий словарь, и они автоматически заработают. Мы принимаем pull requests.
Ограничений со стороны OinkTech нет — это статичные файлы. Ограничения могут накладывать GitHub (raw.githubusercontent.com) или GitHub Pages, но они достаточно высоки для большинства проектов. Для продакшн-нагрузки рекомендуем форкнуть репо и деплоить данные в CDN или Cloudflare Pages.
Парсер использует три уровня источников: XMLTV-агрегаторы, Яндекс.ТВ API и веб-поиск (DuckDuckGo). Если канал не найден ни в одном — в JSON-файле канала будет пустой массив programs. Парсер никогда не генерирует фиктивные данные-заглушки. Добавьте канал в реестр CHANNEL_REGISTRY с правильными алиасами.
Достаточно одного fetch-запроса к JSON-файлу нужного канала и рендеринга результата. Никаких SDK, npm-пакетов или iframe не нужно. Смотри пример кода в разделе «Интеграция».
Весь сервис работает исключительно на GitHub: Actions для парсинга, репозиторий для хранения данных, Pages для лендинга. Это означает нулевые расходы на хостинг и максимальную надёжность. После форка ты можешь деплоить на Vercel, Netlify, Cloudflare Pages или Tatnet.

Условия использования и политика

📋 Условия использования

Используя OinkTech API, вы соглашаетесь со следующими условиями:

  • API предоставляется бесплатно «как есть» (as-is) без гарантий доступности
  • Использование в коммерческих проектах разрешено при указании источника
  • Запрещено использование для генерации спама или массовых рассылок
  • Мы оставляем право изменить структуру данных с предупреждением в README
  • Автоматические запросы должны соблюдать разумные интервалы (не чаще раза в минуту)
  • Данные парсятся из открытых источников и предоставляются для удобства разработчиков
  • OinkTech не несёт ответственности за точность данных третьих сторон

🔒 Политика конфиденциальности

OinkTech API не собирает персональные данные пользователей:

  • Мы не используем cookies и не устанавливаем трекеры на лендинге
  • Данные о запросах к raw.githubusercontent.com обрабатываются GitHub согласно их политике
  • Парсер не сохраняет персональные данные из источников
  • GitHub Actions логи содержат только технические метаданные (время запуска, количество найденных записей)
  • Лендинг размещён на GitHub Pages — применяется политика конфиденциальности GitHub
  • Мы не продаём и не передаём никакие данные третьим лицам
